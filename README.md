# CyBeer

Встраиваемый таймер «пивная олимпиада» на **ESP32-C3**: дисплей TM1637, кнопка/концевик, адресная лента WS2812, веб-интерфейс и турнирная сетка в прошивке.

## Компоненты

| Компонент        | Назначение                          |
|-----------------|-------------------------------------|
| ESP32-C3        | MCU, Wi-Fi STA/AP, HTTP, LittleFS    |
| TM1637 (4×7-сегм.) | Показ времени заезда / нулей в PREP |
| Кнопка / концевик | Старт/стоп таймера (через GPIO)     |
| WS2812 (лента) | Состояния: готовность, заезд, финиш, ожидание claim, подиум |
| АЦП / делитель (опц.) | Оценка уровня батареи (`batteryPercent` в API) |

## GPIO (см. `firmware/components/cybeer_config/include/cybeer_config.h`)

| Сигнал           | GPIO | Примечание        |
|------------------|------|-------------------|
| Переключатель    | 9    | Вход с подтяжкой в прошивке/схеме по спецификации |
| TM1637 CLK       | 6    |                   |
| TM1637 DIO       | 7    |                   |
| WS2812 DATA      | 8    | RMT-драйвер       |
| Батарея (ADC)    | 0    | Канал АЦП по конфигу |

Подробнее — [docs/wiring.md](docs/wiring.md) (полная распиновка и таблицы подключения), [docs/flashing.md](docs/flashing.md) (прошивка).

## Схема подключения (обзор)

```
                    +------------------+
        TM1637 CLK -|GPIO6         GPIO8|--- WS2812 DIN
        TM1637 DIO -|GPIO7         GPIO9|--- Switch / кнопка
                    |              GPIO0|--- Battery sense (ADC)
                    |             Wi-Fi  |--- антенна / сеть
                    +------------------+
                            ESP32-C3
```

Питание 3.3 V для MCU и логики; для ленты — по даташиту WS2812 (часто 5 V и уровень данных через преобразователь/делитель при необходимости).

## Сборка и прошивка

Кратко: `cd firmware && idf.py set-target esp32c3 && idf.py build && idf.py -p <PORT> flash monitor`.

Полная инструкция (Windows/Linux, разделы, LittleFS, OTA, типичные ошибки) — **[docs/flashing.md](docs/flashing.md)**.  
Разметка разделов и **OTA по сети** — [firmware/README.md](firmware/README.md).

### Образ LittleFS (веб-статика)

Статика копируется в раздел `littlefs` (путь монтирования в коде: `/littlefs`, веб-корень `www`). Способ прошивки раздела зависит от вашего `partitions.csv` и скриптов проекта: соберите образ LittleFS с содержимым `firmware/frontend_dist/www` и прошейте его в раздел с меткой `littlefs` (например `esptool.py write_flash ...` или цель IDF, если добавлена).

## Frontend (веб-интерфейс)

Исходники фронтенда лежат в `frontend/`. При сборке прошивки CMake автоматически синхронизирует файлы:

```
frontend/ → firmware/frontend_dist/www/ → LittleFS image → flash partition "littlefs"
```

### Обновление фронтенда

1. Отредактируйте файлы в `frontend/`
2. Пересоберите и прошейте:
   ```bash
   cd firmware
   idf.py build flash
   ```
   LittleFS-образ пересоздаётся автоматически при каждой сборке.

## Первичная настройка Wi-Fi и mDNS

1. При отсутствии сохранённых учётных данных устройство поднимает режим настройки (AP) — подключитесь к нему по инструкции прошивки (`cybeer_wifi`).
2. Передайте SSID/пароль своей сети через предусмотренный в прошивке способ (веб-форма настройки / API — см. код `cybeer_wifi`).
3. В STA после подключения устройство доступно по IP из `GET /api/status` (`wifi.staIp`) и по **mDNS**: хост задаётся **`CYBEER_MDNS_HOSTNAME`** (`cybeer` → `http://cybeer.local`, если клиент поддерживает mDNS).

## Чемпионат: поток PREP → подъём → установка → claim

1. **PREP** — дисплей нули, лента в режиме «вооружён» (готов к старту).
2. **Lift** — первое нажатие/смена концевика: старт таймера, **RUNNING**, побег по ленте.
3. **Place** — повторное нажатие: **FINISHED**, на дисплее время заезда; короткая вспышка на ленте, затем при наличии **незаклеймленного** заезда — маяк **ожидания claim** (пока заезд не заклеймлен).
4. **Claim** — участник на телефоне открывает страницу claim, закрепляет заезд за собой (`POST /api/runs/:id/claim`). При особом результате (топ‑3 среди заклеймленных или личный рекорд) — эффект **подиума** на ленте. Турнирная логика обновляется (`cybeer_tournament`).

## Админка: PIN и число светодиодов

- Перед операциями **экспорта**, **сброса данных**, ручного ввода заездов, турниров и **OTA** нужен админ‑PIN: заголовок **`X-Admin-Pin`**. PIN задаётся один раз (`POST /api/admin/pin/setup`); в NVS хранится **соль и SHA‑256(salt ‖ pin)**.
- **Количество светодиодов** и яркость задаются через **`PUT /api/settings`** (с PIN) и отражаются в **`GET /api/status`** (`ledCount`, `ledBrightness`). После смены настроек прошивка перезагружается.
