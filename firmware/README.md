# CyBeer — прошивка ESP32-C3

## OTA (обновление по воздуху)

Разметка флеша задаётся в `partitions.csv`: два слота приложения `ota_0` / `ota_1` по **0x140000** байт каждый.

### Подтверждение успешной загрузки

После старта прошивка вызывает `esp_ota_mark_app_valid_cancel_rollback()`, если образ был в состоянии «ожидает проверки», чтобы отменить откат на предыдущую версию.

### Админ-пин

Эндпоинты `/api/admin/ota/*` требуют заголовок **`X-Admin-Pin`**. PIN задаётся через `POST /api/admin/pin/setup` (один раз). В NVS (`namespace cybeer`) хранятся **соль** (`admin_pin_salt`, 16 байт) и **SHA-256(salt ‖ pin)** (`admin_pin_hash`, 32 байта). Проверка выполняется через **`cybeer_admin_verify_pin`** — тот же алгоритм, что и для остальных админ-операций в `cybeer_web`. До настройки PIN OTA вернёт **403**. Отправляемый в заголовке PIN — в открытом виде (до `CYBEER_ADMIN_PIN_MAX_LEN` символов).

### Обновление по HTTPS URL

```http
POST /api/admin/ota/url
X-Admin-Pin: <pin>
Content-Type: application/json

{"url":"https://example.com/firmware.bin"}
```

Загрузка выполняется фоновой задачей (`esp_https_ota` с системным CA bundle). Ответ **202** с `{"ok":true,"status":"started"}`. После успешного OTA устройство перезагружается само.

Разрешены только URL с префиксом `https://`.

### Загрузка образа телом запроса

```http
POST /api/admin/ota/upload
X-Admin-Pin: <pin>
Content-Type: application/octet-stream
Content-Length: <размер ≤ 0x140000>

<сырой .bin>
```

Допустим также **`multipart/form-data`** с одним файлом (первое поле после заголовков части — двоичный образ). Максимальный размер тела запроса — **0x140000** байт.

После проверки образа выставляется следующий загрузочный раздел и выполняется перезагрузка. Успешный ответ: `{"ok":true}`.

### Одновременные операции

Параллельно может выполняться только одна операция OTA; при занятости вернётся **409** `{"error":"ota busy"}`.
