# CyBeer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** ESP-IDF firmware + embedded web app for a beer-championship stopwatch with TM1637, WS2812B, switch-driven state machine, leaderboard, tournaments, WebSocket live timer, OTA, and battery status.

**Architecture:** Modular IDF components (`cybeer_*`) around a central `cybeer_app` state machine. NVS for settings/Wi-Fi/PIN; LittleFS JSON for domain data. `esp_http_server` serves static UI from LittleFS and REST; `httpd_ws` broadcasts state/timer. Wi-Fi AP+STA with captive `/setup`.

**Tech Stack:** ESP-IDF 5.2+, ESP32-C3, `led_strip`, `esp_timer`, LittleFS, cJSON, mbedTLS (OTA), mDNS.

**Spec:** `docs/superpowers/specs/2026-05-17-cybeer-design.md`

void cybeer_fsm_reset_to_prep(void);

- [ ] **Step 2: FSM test — full cycle**

```c
void test_prep_running_finished_ready_prep(void)
{
    cybeer_fsm_init(&(cybeer_fsm_callbacks_t){0});
    cybeer_fsm_reset_to_prep();
    TEST_ASSERT_EQUAL(CYBEER_STATE_PREP, cybeer_fsm_snapshot().state);
    cybeer_fsm_on_switch(false, 1000000); /* lift */
    TEST_ASSERT_EQUAL(CYBEER_STATE_RUNNING, cybeer_fsm_snapshot().state);
    cybeer_fsm_on_switch(true, 5000000); /* place */
    TEST_ASSERT_EQUAL(CYBEER_STATE_FINISHED, cybeer_fsm_snapshot().state);
    cybeer_fsm_on_switch(false, 6000000); /* remove */
    TEST_ASSERT_EQUAL(CYBEER_STATE_READY, cybeer_fsm_snapshot().state);
    cybeer_fsm_on_switch(true, 8000000); /* return */
    TEST_ASSERT_EQUAL(CYBEER_STATE_PREP, cybeer_fsm_snapshot().state);
}
```

Implement transitions exactly per spec §4.2. Ignore switch edges unless `pressed_stable` changed.
